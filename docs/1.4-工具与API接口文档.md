# 1.4 工具/API 接口文档

## 1. HTTP API 总览

服务入口：FastAPI。

默认本地地址：

```text
http://127.0.0.1:8000
```

前端 Vite 代理：

```text
http://localhost:5173 -> http://127.0.0.1:8000
```

## 2. 基础接口

### 2.1 健康检查

```http
GET /health
```

响应：

```json
{"status": "ok"}
```

### 2.2 根路径

```http
GET /
```

用途：查看应用名称、版本、阶段和 Swagger 地址。

### 2.3 配置摘要

```http
GET /config
```

用途：查看当前环境、Chroma、模型、MCP、LangSmith 和 API Key 是否配置。

注意：接口只返回 key 是否设置，不返回密钥明文。

## 3. Chat API

### 3.1 非流式问答

```http
POST /api/chat
Content-Type: application/json
```

请求：

```json
{
  "question": "什么是 ITPM 策略？",
  "session_id": "可选，会话 ID",
  "user_id": "default"
}
```

响应：

```json
{
  "answer": "回答正文",
  "citations": ["4.2.2"],
  "score": null,
  "session_id": "服务端会话 ID"
}
```

行为：

- 如果没有传 `session_id`，后端创建新会话。
- 如果传入已有 `session_id`，后端加载最近 N 轮消息作为短期上下文。
- 本轮问答结束后写入 MySQL `agent_conversation` / `agent_message` 会话表。

### 3.2 SSE 流式问答

```http
POST /api/chat/stream
Content-Type: application/json
Accept: text/event-stream
```

请求：

```json
{
  "question": "统计各专业的事务数量",
  "session_id": "可选，会话 ID",
  "user_id": "default"
}
```

SSE 事件：

| 事件 | 数据字段 | 含义 |
| --- | --- | --- |
| `token` | `text` | LLM 输出文本 chunk |
| `tool_call` | `name`, `args` | LLM 或 Supervisor 决定调用工具 |
| `tool_result` | `name`, `preview` | 工具执行结果预览 |
| `meta` | `citations`, `session_id`, `short_term_count`, `chart_data_uri` 等 | 尾部元数据 |
| `done` | `{}` | 流结束 |
| `error` | `message` | 异常信息 |

前端注意：

- 浏览器原生 `EventSource` 不支持 POST body，所以前端使用 `fetch` 读取 stream。
- SQL 分支可能不产生 token，前端会 fallback 到 `/api/chat` 获取完整答案。

## 4. Session API

### 4.1 会话列表

```http
GET /api/sessions?user_id=default&limit=30
```

响应字段：

| 字段 | 含义 |
| --- | --- |
| `id` | 会话 ID |
| `user_id` | 用户 ID |
| `title` | 会话标题 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |
| `message_count` | 消息数 |

### 4.2 会话详情

```http
GET /api/sessions/{session_id}?user_id=default
```

响应：

```json
{
  "session": {},
  "messages": []
}
```

### 4.3 删除会话

```http
DELETE /api/sessions/{session_id}?user_id=default
```

如果传入 `user_id`，后端会先校验该会话属于当前用户，再执行删除；不属于该用户时返回 404。

响应：

```json
{"deleted": true}
```

## 7. MCP 接口

当前 MCP Server 使用官方 `mcp` SDK 的 FastMCP 暴露工具。

支持：

- stdio。
- streamable-http。
- 通过 `MCP_ENABLED` 开关切换本地工具和 MCP 工具。

当前 MCP 暴露工具与 LangChain 工具保持同类能力：换算、条款查询、文档检索和规格检索。

## 8. 前端接口调用

前端服务文件：

```text
frontend/src/services/chatStream.js
```

核心行为：

1. `checkHealth()` 调用 `/health`。
2. `streamChat()` 调用 `/api/chat/stream`。
3. `sendChat()` 调用 `/api/chat`。
4. `login()` 调用 `/api/auth/login`。
5. `getMe()` 调用 `/api/auth/me` 解析本地 token。
6. 请求体携带 `session_id` 和 `user_id`。
7. 收到 `meta.session_id` 后保存当前会话。
8. `listSessions()` 调用 `/api/sessions`。
9. `getSession()` 调用 `/api/sessions/{session_id}`。
10. `deleteSession()` 调用 `/api/sessions/{session_id}` 并携带 `user_id`。

## 9. 登录与权限接口

当前已实现基础登录接口。登录优先读取当前数据库中的 `user` 表；若本地开发环境无法连接 MySQL，则允许使用 `admin/admin` 和 `engineer/engineer` 作为非生产演示账号。

用户表字段：

| 字段 | 类型 | 含义 |
| --- | --- | --- |
| `user_id` | int | 用户主键 |
| `username` | varchar(255) | 账号名 |
| `password` | varchar(255) | 账号密码 |
| `authority_flag` | int | `0=engineer`，`1=admin` |

已实现 API：

```http
POST /api/auth/login
GET /api/auth/me
```

当前阶段不实现后端 logout，前端清理 localStorage 中的本地 token 即可。

登录成功后应返回用户信息和角色：

```json
{
  "token": "local-demo-token",
  "user": {
    "user_id": "1",
    "username": "admin",
    "role": "admin",
    "authority_flag": 1,
    "permissions": ["用户管理"]
  }
}
```

当前 token 是前端本地演示 token，不是 JWT，也不作为生产级鉴权凭据。

## 10. 评估结果接口

### 10.1 最新 Golden Set 评估摘要

```http
GET /api/evaluation/latest
```

用途：读取本地 `data/eval_results/golden_eval_summary.json`，返回管理员评估看板需要的聚合摘要。该接口不返回原始 Golden Set 问题、不返回完整 SQL、不返回完整检索片段，仅返回数据集画像、合约指标、Prediction 指标和场景分布。

默认评估文件路径可通过环境变量覆盖：

```env
EVAL_RESULTS_PATH=data/eval_results/golden_eval_summary.json
```

如果接口返回 404，说明当前环境还没有生成评估摘要，需要先运行：

```powershell
uv run python scripts/eval_golden_set.py
uv run python scripts/replay_golden_set.py --mode oracle --evaluate
```

响应示例：

```json
{
  "title": "Golden Set 评估摘要",
  "generatedAt": "2026-06-24 15:27",
  "source": "data/eval_results/golden_eval_summary.json",

    "turns": 390,
```

前端行为：管理员工作台优先调用该接口；当后端不可用或评估文件不存在时，前端回退到 `frontend/src/data/evaluationSummary.js` 中的静态摘要，保证演示页面可用。

### 10.2 Golden Set 失败与风险样例

```http
GET /api/evaluation/failures?limit=8
```

用途：读取本地 `data/eval_results/predictions.jsonl`，按照 prediction 状态、SQL 生成与校验结果、RAG 检索引用、回答是否为空、延迟等规则，返回失败与高风险样例摘要。该接口用于管理员评估看板的“失败与风险样例”列表。

默认 prediction 文件路径可通过环境变量覆盖：

```env
EVAL_PREDICTIONS_PATH=data/eval_results/predictions.jsonl
```

隐私边界：

- 不返回完整 SQL，只返回是否存在、是否通过校验、涉及表名摘要和 SQL 长度。
- 不返回完整问题和完整答案，只返回截断摘要。
- 不返回完整检索片段，只返回数量和来源文件摘要。
- 用于本地演示和管理员排障；生产环境仍应叠加鉴权、审计和脱敏策略。

响应示例：

### 10.3 Golden Set 评估运行历史

```http
GET /api/evaluation/runs?limit=10
```

用途：扫描 `EVAL_RESULTS_PATH` 所在目录中的评估摘要 JSON，返回最近评估运行批次、数据集规模、核心指标、产物存在状态和 LangSmith 查询线索。该接口用于管理员看板的“评估运行历史”区域。

返回内容说明：

| 字段 | 含义 |
| --- | --- |
| `runs[].status` | `scored` 表示包含 prediction 指标，`profile-only` 表示仅做 Golden Set 画像 |
| `runs[].artifacts` | summary / markdown / predictions 三类本地产物是否存在 |
| `runs[].traceHint` | LangSmith 查询提示，包含 project、sessionId/query 和说明，不包含真实 trace 内容 |
| `runs[].metrics` | Prediction 数量、SQL 校验率、SQL 表召回、RAG Recall@5 |

## 11. 长期记忆接口

Phase 6.1 新增长期记忆显式管理接口，当前用于数据模型验证和后续前端记忆管理页接入。自动抽取和 Agent 召回尚未接入。

### 11.1 创建长期记忆

```http
POST /api/memory
```

请求示例：

```json
{
  "user_id": "2",
  "memory_type": "preference",
  "content": "用户常看炼油一部的动设备任务",
  "source": "manual",
  "confidence": 0.9,
  "metadata": {
    "department": "炼油一部"
  },
  "actor_id": "admin"
}
```

### 11.2 查询长期记忆

```http
GET /api/memory?user_id=2&status=active&memory_type=preference&limit=50
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `status` | `active` / `disabled` / `deleted` / `all` |
| `memory_type` | 可选过滤条件，如 preference、business_context、query_filter |
| `limit` | 默认 50，最大由后端下钳到 200 |

### 11.3 更新、禁用、软删除

```http
PATCH /api/memory/{memory_id}
POST /api/memory/{memory_id}/disable
DELETE /api/memory/{memory_id}?actor_id=admin&reason=用户要求删除
```

说明：

- 禁用会将 `status` 设置为 `disabled`，后续召回阶段不应使用。
- 删除采用软删除，将 `status` 设置为 `deleted`。
- 每次创建、更新、禁用、删除都会写入 `memory_event`。

### 11.4 查看审计事件

```http
GET /api/memory/{memory_id}/events
```

返回 `created`、`updated`、`disabled`、`deleted`、`accessed` 等事件。当前 Phase 6.1 已实现生命周期事件，`accessed` 会在后续召回阶段接入。
