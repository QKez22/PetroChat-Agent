# 1.6 评测标准文档

## 1. 评测目标

评测体系用于回答：

1. RAG 是否召回了正确条款。
2. 答案是否忠实于检索上下文。
3. NL2SQL 是否生成安全且正确的 SQL。
4. 会话短期记忆是否提升多轮问答效果。
5. 前端是否稳定展示流式输出、图表、引用和错误。
6. 改动后是否出现质量回退。

当前评测重点集中在四条主线：

| 主线 | 评测重点 |
| --- | --- |
| Agent 调用流程 | supervisor 路由、工具调用顺序、权限边界、高风险操作拦截 |
| 记忆管理 | 多轮条件继承、覆盖、追加、清空、应使用/应忽略记忆 |
| RAG 向量检索 | 召回章节、来源文档、关键词覆盖、证据忠实性 |
| Agent 调用结果指标 | SQL 正确性、检索正确性、回答完整性、报告草稿质量、工具状态 |

## 3. RAG 评测

### 3.1 检索指标

| 指标 | 定义 | 建议目标 |
| --- | --- | --- |
| Recall@K | 标注相关 chunk 是否出现在 Top-K | Recall@5 >= 0.85 |
| MRR | 第一个正确 chunk 的倒数排名 | >= 0.65 |
| nDCG@K | 多相关 chunk 的排序质量 | nDCG@5 >= 0.75 |
| Context Precision | 召回上下文中有效片段比例 | >= 0.70 |

### 3.2 生成指标

| 指标 | 定义 | 建议目标 |
| --- | --- | --- |
| Answer Relevance | 是否回答问题 | >= 4/5 |
| Faithfulness | 是否由上下文支持 | >= 4/5 |
| Citation Correctness | 引用是否对应真实条款 | >= 0.90 |
| Hallucination Rate | 无依据内容比例 | <= 0.05 |

回放脚本模式：

| 模式 | 命令 | 用途 |
| --- | --- | --- |
| `oracle` | `uv run python scripts/replay_golden_set.py --mode oracle --evaluate` | 不调用模型，用 Golden Set 期望生成 prediction JSONL，验证评估管道 |
| `agent` | `uv run python scripts/replay_golden_set.py --mode agent --limit 5 --evaluate` | 调用真实 LangGraph Agent，依赖 LLM、Chroma、MySQL 在线 |

Phase 7.1 默认安全策略：

- `--mode agent` 未显式传 `--limit` 时默认只跑 5 轮，避免误触发全量 390 轮模型调用。
- 支持 `--scenario-type` 过滤场景，例如只跑 `nl2sql_condition_memory`。
- 支持重复传入 `--dialogue-id` 精确回放指定对话。
- 支持 `--eval-user-id` 指定数值用户 ID，用于长期记忆隔离。
- agent 回放额外生成 summary JSON，默认路径为 prediction 文件同名 `.summary.json`。

当前脚本已覆盖：

| 模块 | 指标 |
| --- | --- |
| 数据集画像 | 对话组数、轮次数、角色分布、场景分布 |
| SQL 合约 | SQL 模板 AST 校验通过率、写操作违规数、SELECT * 违规数、期望表分布 |
| Memory 合约 | 需要继承的轮次、需要忽略的轮次、澄清轮次、记忆字段规模 |
| RAG 合约 | 证据行数、关键词数量、必须覆盖要点、来源文件分布 |
| Prediction 可选评估 | Agent 成功率、错误数、平均延迟、SQL 校验率、SQL 表召回、SQL 过滤值召回、SQL 合约准确率、SQL 执行准确率、RAG Recall@5、RAG MRR、证据覆盖率、忠实性代理指标、Memory Hit Rate |

失败与风险样例规则：

| 风险来源 | 判定规则 | 等级 |
| --- | --- | --- |
| Agent 调用异常 | `status != ok` | 失败 |
| SQL 路由缺失 SQL | `route=sql` 但 prediction 中没有 SQL | 失败 |
| SQL 安全校验失败 | `sqlglot` AST 校验不通过或出现非 SELECT | 失败 |
| RAG/QA 缺检索引用 | `route=qa/rag` 但 retrieved 为空 | 风险 |
| 空回答 | answer 为空 | 风险 |
| 高延迟 | `latency_ms >= 15000` | 风险 |

Phase 7.3 已增加失败归因基础版。当前不调用额外 LLM，而是基于 prediction JSONL 的结构化字段做规则归因，输出 `attributionSummary`、`severitySummary`、`primaryAttribution` 和 `attributions`。归因类型包括：

| 归因类型 | 触发信号 | 排查入口 |
| --- | --- | --- |
| `system` | prediction status 非 ok | 回放日志、SSE meta、LangSmith trace |
| `routing` | route 缺失或不在已知枚举 | Supervisor 路由提示词和 LangGraph state |
| `sql` | SQL 分支未生成 SQL 或 SQL AST 校验失败 | NL2SQL generator、validator、schema prompt |
| `retrieval` | QA/RAG 分支 retrieved 为空 | Chroma、query rewrite、chunk 切分 |
| `memory` | 记忆场景未记录 memory_used | 短期滑动窗口、长期记忆召回 |
| `output` | answer 为空 | 最终回答汇总节点、流式输出、fallback |
| `performance` | latency_ms 超过阈值 | 工具调用、MySQL 慢查询、LLM 首 token |

运行历史口径：

| 字段 | 说明 |
| --- | --- |
| `status=scored` | 该评估运行包含 prediction 指标，可用于质量趋势对比 |
| `status=profile-only` | 该评估运行仅包含 Golden Set 画像和合约统计 |
| `traceHint` | LangSmith 查询线索，用于定位真实 agent 回放的 session/run，不保存真实 trace 内容 |
| `artifacts` | 记录 summary、markdown、predictions 三类本地产物是否存在 |
