# 1.2 用户需求文档

## 1. 目标用户

| 用户类型 | 主要诉求 | 当前支持程度 |
| --- | --- | --- |
| 一线工程师 | 快速查询规范、条款、事务和任务 | 已实现核心能力 |
| 设备/专业管理人员 | 统计事务和任务执行情况，生成报表 | 已实现 NL2SQL + 报表 |
| 知识管理人员 | 维护规范知识库和评估问答质量 | 部分实现，评估体系待完善 |
| 管理员/面试演示者 | 查看 Agent 路由、工具调用、错误和会话 | 前端本地观测已实现，后端观测待增强 |
| 开发者 | 调试 RAG、SQL、工具和 SSE 链路 | 已有测试和 LangSmith 支持 |

## 2. 用户目标

用户希望通过自然语言完成以下任务：

1. 查询石化规范概念、流程和条款。
2. 查询事务和任务业务数据。
3. 生成统计表和图表。
4. 进行单位换算和辅助计算。
5. 在多轮对话中延续上下文。
6. 查看回答依据、工具调用和错误信息。

## 5. 典型用户故事

### 用户故事 1：规范问答

作为一线工程师，我希望输入“什么是 ITPM 策略？”，系统能从规范知识库中检索相关条款，并给出带引用的解释。

验收标准：

- 返回自然语言答案。
- 答案包含规范条款引用。
- 前端展示引用信息。

### 用户故事 2：事务查询

作为设备管理人员，我希望输入“查询仪表专业的运行中事务清单”，系统能识别“仪表专业=仪”，生成安全 SQL，并返回事务列表。

验收标准：

- SQL 查询 `affair` 表。
- `specialty='仪'`。
- `trigger_status=1`。
- 只执行 SELECT。

### 用户故事 3：报表分析

作为管理者，我希望输入“统计各专业的事务数量”，系统能返回表格并生成图表。

验收标准：

- 返回 Markdown 表。
- SSE `meta` 中带 `chart_data_uri`。
- 前端显示图表。

### 用户故事 4：多轮会话

作为用户，我希望先问“查询仪表专业的运行中事务”，再追问“按执行部门统计一下”，系统能理解第二问继承上一轮上下文。

验收标准：

- 前端复用同一 `session_id`。
- 后端加载最近 N 轮历史。
- Prompt 中包含上一轮问答。
