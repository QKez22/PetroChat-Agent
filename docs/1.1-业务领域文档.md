# 1.1 业务领域文档

## 1. 项目定位

PetroChat-Agent 是面向石化领域的智能问答与多 Agent 数据分析平台。项目以石化规范文档和事务/任务业务库为核心数据源，提供规范问答、业务数据查询、单位换算、条款检索、报表生成和多轮会话能力。

项目不是通用聊天机器人，而是一个垂直领域 LLM 应用工程示例，重点展示：

- 石化规范知识库 RAG。
- Tool Calling 与领域工具。
- MCP Server 工具标准化暴露。
- Supervisor 多 Agent 路由。
- 安全 NL2SQL 与报表输出。
- 会话持久化与短期滑动窗口。

## 2. 业务背景

石化企业存在大量规范、制度、设备完整性文件、事务定义和任务执行数据。传统检索和报表查询方式通常存在以下问题：

| 问题 | 表现 | PetroChat-Agent 的对应能力 |
| --- | --- | --- |
| 规范文档长且分散 | 工程师难以快速定位条款 | RAG 检索 + 引用章节 |
| 业务数据字段复杂 | 事务和任务容易混淆 | Schema 增强 + NL2SQL few-shot |
| 数据查询门槛高 | 需要手写 SQL | 自然语言转 SQL |
| 报表展示不直观 | 查询结果需要二次处理 | Markdown 表 + 图表 data URI |
| Agent 过程不可见 | 不知道模型为何调用工具 | SSE 工具事件 + 管理员观测 |

### 2.1 用户角色

当前系统角色暂定为两类：

| 角色 | 标识 | 业务定位 |
| --- | --- | --- |
| 管理员 | `admin` | 系统管理、知识库管理、配置管理、审计与评估 |
| 工程师用户 | `engineer` | 提出业务问题、查询知识和数据、生成回答/报告、反馈质量 |

登录账号可直接使用当前数据库中的 `user` 表，角色由 `authority_flag` 映射：

| 字段 | 类型 | 含义 |
| --- | --- | --- |
| `user_id` | int | 用户主键 |
| `username` | varchar(255) | 账号名 |
| `password` | varchar(255) | 账号密码 |
| `authority_flag` | int | `0=engineer`，`1=admin` |

## 3. 核心业务对象

### 3.1 规范知识

规范知识来自石化领域 Word 文档，经过解析、切片、Embedding 后入 Chroma 向量库。

当前能力：

- 使用 `python-docx` 解析 `.docx`。
- 优先使用 Word `outlineLvl`，并用章节编号正则兜底。
- 每个 chunk 带有 `source_doc`、`section_number`、`section_path`、`doc_code` 等元数据。
- 回答中可生成 `[4.2.2]` 形式的条款引用。

### 3.2 事务

事务对应业务库中的 `affair` 表。它表示周期性或规则性的业务事项定义，例如检查巡检、维护活动、会议总结、执行确认等。

关键业务规则：

- 用户说“事务”时，优先查询 `affair` 表。
- `trigger_status` 表示事务运行状态：`0=停止`，`1=运行`，`-1=删除`。
- `ticket_flag`、`check_flag` 是反直觉字段：`0=是`，`1=否`。
- `specialty` 是专业枚举，常见值为 `综合/动/静/电/仪/其它`，口语“仪表专业”应映射为 `仪`。

### 3.3 任务

任务对应业务库中的 `affair_task` 表。它表示事务下具体执行任务，通常与设备、执行角色、提报人、提报时间等字段相关。

关键业务规则：

- 用户说“任务/执行情况/巡检/检修/设备维度”时，通常需要查询 `affair_task`。
- `affair.affair_id` 与 `affair_task.associated_affair_id` 是关联键。
- 不应使用两张表的自增 `id` 做跨表关联。

### 3.4 会话

会话用于保存用户与 Agent 的对话历史。当前实现使用 MySQL 应用表：

- `agent_conversation`：会话基本信息、所属用户、删除状态和保留策略。
- `agent_message`：用户和助手消息。
- 前端保存当前 `session_id`。
- 后端按最近 N 轮加载短期滑动窗口。

## 4. 当前业务场景

| 场景 | 示例问题 | 路由 |
| --- | --- | --- |
| 规范问答 | 什么是 ITPM 策略？ | `qa` |
| 条款查询 | 高桥石化备品配件管理细则 1.4.1 写了什么？ | `general` + `lookup_section` |
| 文档内检索 | 在设备完整性文件里查设备分级 | `general` + `search_within_doc` |
| 事务查询 | 查询仪表专业的运行中事务清单 | `sql` |
| 任务统计 | 统计每个事务下的任务数量及上传情况 | `sql` |
| 报表输出 | 统计各专业的事务数量 | `sql` + report |
| 单位换算 | 1 MPa 等于多少 psi？ | `general` + `convert_unit` |
| 多轮追问 | 上一个结果里按部门再汇总 | `session_id` + short-term memory |

## 5. 业务边界

当前支持：

- 石化规范类问答。
- 事务/任务业务数据查询。
- 业务数据统计和图表。
- 常用石化单位换算。
- 短期多轮上下文。
- 前端登录页和 admin/engineer 基础角色入口。

当前不支持或暂未完成：

- 服务端完整 RBAC 拦截。
- 长期记忆管理页。
- 自动评估集看板。
- 完整 Docker 一键演示。
- 对所有业务表开放 NL2SQL 查询。
- 写入、修改、删除业务数据库。
