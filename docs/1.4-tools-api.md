# 1.4 工具/API 接口文档

## 1. HTTP API 总览

服务入口：FastAPI。

默认本地地址：

```text
http://127.0.0.1:8000
```

前端 Vite 代理：

```text
http://localhost:5173 -> http://127.0.0.1:8000
```

## 2. 基础接口

### 2.1 健康检查

```http
GET /health
```

响应：

```json
{"status": "ok"}
```

### 2.2 根路径

```http
GET /
```

用途：查看应用名称、版本、阶段和 Swagger 地址。

### 2.3 配置摘要

```http
GET /config
```

用途：查看当前环境、Chroma、模型、MCP、LangSmith 和 API Key 是否配置。

注意：接口只返回 key 是否设置，不返回密钥明文。

## 3. Chat API

### 3.1 非流式问答

```http
POST /api/chat
Content-Type: application/json
```

请求：

```json
{
  "question": "什么是 ITPM 策略？",
  "session_id": "可选，会话 ID",
  "user_id": "default"
}
```

响应：

```json
{
  "answer": "回答正文",
  "citations": ["4.2.2"],
  "score": null,
  "session_id": "服务端会话 ID"
}
```

行为：

- 如果没有传 `session_id`，后端创建新会话。
- 如果传入已有 `session_id`，后端加载最近 N 轮消息作为短期上下文。
- 本轮问答结束后写入 SQLite 会话库。

### 3.2 SSE 流式问答

```http
POST /api/chat/stream
Content-Type: application/json
Accept: text/event-stream
```

请求：

```json
{
  "question": "统计各专业的事务数量",
  "session_id": "可选，会话 ID",
  "user_id": "default"
}
```

SSE 事件：

| 事件 | 数据字段 | 含义 |
| --- | --- | --- |
| `token` | `text` | LLM 输出文本 chunk |
| `tool_call` | `name`, `args` | LLM 或 Supervisor 决定调用工具 |
| `tool_result` | `name`, `preview` | 工具执行结果预览 |
| `meta` | `citations`, `session_id`, `short_term_count`, `chart_data_uri` 等 | 尾部元数据 |
| `done` | `{}` | 流结束 |
| `error` | `message` | 异常信息 |

前端注意：

- 浏览器原生 `EventSource` 不支持 POST body，所以前端使用 `fetch` 读取 stream。
- SQL 分支可能不产生 token，前端会 fallback 到 `/api/chat` 获取完整答案。

## 4. Session API

### 4.1 会话列表

```http
GET /api/sessions?user_id=default&limit=30
```

响应字段：

| 字段 | 含义 |
| --- | --- |
| `id` | 会话 ID |
| `user_id` | 用户 ID |
| `title` | 会话标题 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |
| `message_count` | 消息数 |

### 4.2 会话详情

```http
GET /api/sessions/{session_id}?user_id=default
```

响应：

```json
{
  "session": {},
  "messages": []
}
```

### 4.3 删除会话

```http
DELETE /api/sessions/{session_id}
```

响应：

```json
{"deleted": true}
```

## 7. MCP 接口

当前 MCP Server 使用官方 `mcp` SDK 的 FastMCP 暴露工具。

支持：

- stdio。
- streamable-http。
- 通过 `MCP_ENABLED` 开关切换本地工具和 MCP 工具。

当前 MCP 暴露工具与 LangChain 工具保持同类能力：换算、条款查询、文档检索和规格检索。

## 8. 前端接口调用

前端服务文件：

```text
frontend/src/services/chatStream.js
```

核心行为：

1. `checkHealth()` 调用 `/health`。
2. `streamChat()` 调用 `/api/chat/stream`。
3. `sendChat()` 调用 `/api/chat`。
4. 请求体携带 `session_id` 和 `user_id`。
5. 收到 `meta.session_id` 后保存当前会话。
