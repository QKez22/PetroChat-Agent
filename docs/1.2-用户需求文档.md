# 1.2 用户需求文档

## 1. 目标用户

| 用户类型 | 主要诉求 | 当前支持程度 |
| --- | --- | --- |
| 管理员 `admin` | 用户、角色、文档、知识库、索引、配置、日志、评估与审计管理 | 已规划，部分观测能力已实现 |
| 工程师用户 `engineer` | 查询规范制度、设备知识、巡检维修隐患作业流程、业务数据和个人历史 | 已实现核心问答能力 |

## 2. 用户目标

工程师用户希望通过自然语言完成以下任务：

1. 查询石化规范概念、流程和条款。
2. 查询事务和任务业务数据。
3. 生成统计表和图表。
4. 进行单位换算和辅助计算。
5. 在多轮对话中延续上下文。
6. 查看回答依据、工具调用和错误信息。

管理员希望完成以下管理任务：

1. 管理用户与角色。
2. 管理文档上传、删除、更新和知识库重建。
3. 管理向量索引、数据库连接、工具/API、提示词和模型参数。
4. 查看所有问答记录、工具调用记录、系统运行日志和质量评估结果。
5. 导出数据、删除异常数据、审核工程师提交的问题反馈。

## 2.1 角色权限清单

### 管理员 admin

| 权限域 | 权限 |
| --- | --- |
| 用户与角色 | 用户管理、角色分配 |
| 文档与知识库 | 文档上传、删除、更新；知识库重建；向量索引管理 |
| 系统配置 | 数据库连接配置、工具/API 配置、Agent 提示词配置、模型参数配置 |
| 观测与审计 | 查看所有问答记录、查看所有工具调用记录、查看系统运行日志、查看质量评估结果 |
| 数据治理 | 导出数据、删除异常数据、审核工程师提交的问题反馈 |

### 工程师用户 engineer

| 权限域 | 权限 |
| --- | --- |
| 问答 | 提出业务问题；查询制度、规范、管理办法；查询设备相关知识 |
| 业务流程 | 查询巡检、维修、隐患、作业流程 |
| Agent 使用 | 查看知识库检索结果；使用 Agent 生成回答；使用 Agent 生成报告草稿 |
| 数据查询 | 查询允许范围内的数据库信息 |
| 历史与反馈 | 查看自己的历史问答记录；对回答进行反馈；标记回答是否正确；提交知识库补充建议 |

## 5. 典型用户故事

### 用户故事 1：规范问答

作为一线工程师，我希望输入“什么是 ITPM 策略？”，系统能从规范知识库中检索相关条款，并给出带引用的解释。

验收标准：

- 返回自然语言答案。
- 答案包含规范条款引用。
- 前端展示引用信息。

### 用户故事 2：事务查询

作为设备管理人员，我希望输入“查询仪表专业的运行中事务清单”，系统能识别“仪表专业=仪”，生成安全 SQL，并返回事务列表。

验收标准：

- SQL 查询 `affair` 表。
- `specialty='仪'`。
- `trigger_status=1`。
- 只执行 SELECT。

### 用户故事 3：报表分析

作为管理者，我希望输入“统计各专业的事务数量”，系统能返回表格并生成图表。

验收标准：

- 返回 Markdown 表。
- SSE `meta` 中带 `chart_data_uri`。
- 前端显示图表。

### 用户故事 4：多轮会话

作为用户，我希望先问“查询仪表专业的运行中事务”，再追问“按执行部门统计一下”，系统能理解第二问继承上一轮上下文。

验收标准：

- 前端复用同一 `session_id`。
- 后端加载最近 N 轮历史。
- Prompt 中包含上一轮问答。
